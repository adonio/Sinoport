# Sinoport OS 第一批前端 Demo 演示脚本

## 1. 演示目标

用 8-12 分钟解释第一批前端 demo 已经完成了什么：

- 平台端已经从 v0.2 菜单升级到 v1.0 主控结构
- 货站端已经从“看表格”升级到“对象 + 状态 + 任务 + 文件门槛”
- PDA 端已经把 `MME` 进港样板链路统一到任务卡模型

## 2. 演示路径

### Step 1: 平台运行态势中心

入口：`/platform/operations`

要点：

- 先看顶部 KPI，说明平台现在以链路和风险为主视角
- 看链路与接口告警，说明这批只做前端 demo，但 contract 已冻结
- 看待处理动作和站点健康度矩阵，说明平台不是传统 ERP 菜单

### Step 2: 平台规则与指令引擎

入口：`/platform/rules`

要点：

- 说明服务等级、任务模板、证据要求
- 重点讲三条门槛：
  - 理货完成前不得发送 NOA
  - POD 双签前不得 Closed
  - Manifest 未冻结不得飞走归档

### Step 3: 货站看板与进港页

入口：`/station/dashboard` -> `/station/inbound`

要点：

- 看货站看板里的阻塞和待复核
- 进入进港管理，说明状态链、文件门槛、阻断提示

### Step 4: Shipment 详情闭环

入口：`/station/shipments`

要点：

- 打开一条 Shipment
- 在详情页说明：
  - 对象摘要
  - 履约时间线
  - 关联任务
  - 关联文件
  - 关联异常
- 演示详情页如何跳到：
  - `/station/documents`
  - `/station/tasks`
  - `/station/exceptions`

### Step 5: 异常中心

入口：`/station/exceptions`

要点：

- 说明异常不再只是列表
- 展示“异常 -> 阻断任务 -> 恢复动作”
- 说明第一批已经把异常和门槛真正连起来

### Step 6: PDA `MME` 样板链路

入口：`/mobile/login` -> `/mobile/inbound`

推荐航班：`SE803`

要点：

- 航班概览页先讲统一任务卡
- 进入理货页，说明扫码、纳入统计、阻断提示、证据要求
- 进入组托页，说明同票同托
- 进入装车计划与装车执行页，说明车牌 / Collection Note / 核对员门槛

## 3. 结束语

结论只讲三句：

1. 第一批已经把平台、货站、PDA 三端的前端 demo 主干连起来了。
2. 当前没有做任何后端实现，但产品结构、状态机、任务流和文件门槛已经可完整演示。
3. 下一阶段才能在这套前端 demo 基线之上接真实后端。
