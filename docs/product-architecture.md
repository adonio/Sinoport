# Sinoport 页面规划

## 1. 本次规划目标

本次项目先落两部分原型：

1. 企业端网站
2. 后台管理页面

页面规划严格围绕商业模型说明书，不按传统物流官网或传统 ERP 菜单思路展开。

## 2. 企业端网站信息架构

### 2.1 目标

让客户、合伙人和合作站点快速理解三件事：

- Sinoport 卖的是承诺兑现，不是单点货运动作
- Sinoport 有端到端控制逻辑，而不是单一机场工具
- Sinoport 正在形成跨区域货站网络

### 2.2 推荐一级结构

1. 首页 `Home`
2. 产品与服务 `Promise-Driven Fulfillment`
3. 履约网络 `Network`
4. 控制系统 `Sinoport OS`
5. 角色入口 `Platform Admin / Station Admin`
6. 项目资料入口 `Docs`

### 2.3 首页模块建议

1. Hero
   说明产品定位、价值主张与核心口号。
2. Why Sinoport
   强调“不只是地服 / 不只是 SaaS / 不只是项目制”。
3. 端到端履约链路
   展示五段链路与关键控制节点。
4. 货站网络
   体现 `URC、KGF、NVI、RZE、MST、BoH、MME` 的区域布局。
5. KPI 与服务等级
   展示 `Timely / Accurate / Visible / Accountable` 与 `P1/P2/P3`。
6. 四层架构
   展示 `ERP / 供应链系统 / OS / 可信数据层`。
7. 项目资料
   提供 PDF 与结构化文档入口，形成内部共识中心。

### 2.4 官网表现重点

- 用“承诺兑现”作为视觉与文案主轴
- 用链路、控制层、节点网络解释业务，不堆砌营销口号
- 七个机场节点必须清晰展示在网站中
- `MME` 要标为样板站或 SOP 样板站

## 3. 后台管理页面规划

后台分两个层级，必须明确分治，不做成一个菜单很深的大后台。

## 4. 平台管理方后台

### 4.1 目标

平台管理方主要做“网络治理”和“规则治理”，不是日常逐票操作。

### 4.2 推荐核心模块

1. 网络总览
   查看站点数、区域分布、控制深度、试点状态。
2. 货站管理
   新增 / 编辑不同地区货站，维护站点编码、区域、控制层级、SLA 能力。
3. 航线与网络配置
   维护主要流向、航线链路与区域覆盖。
4. 主数据治理
   统一 `Order / Shipment / AWB / Truck / Flight / POD / Contract / Event`。
5. 服务等级与规则中心
   维护 `P1/P2/P3`、硬门槛节点、异常分类字典、SLA 阈值。
6. 接口治理
   管理 `ERP`、供应链系统、航班状态、末端回传、`POD` 接口状态。
7. 审计与可信预留
   查看 `Event ID / Event Hash / Signature Ref / Notarization Ref` 预留状态。

### 4.3 首页布局建议

- 顶部是全网 KPI
- 左侧是区域网络与货站台账
- 右侧是新增货站表单或待审批变更
- 下方是规则、接口、服务等级、试点推进

## 5. 货站管理员后台

### 5.1 目标

货站管理员主要做“站内执行控制”，要能把 SOP 变成状态机。

### 5.2 推荐核心模块

1. 班次与航班看板
   查看落地、拆板、复核、装车、POD 等状态。
2. 货件操作板
   按 `AWB / Shipment / Truck / Priority` 处理任务。
3. SOP 节点控制
   把站点规则转成硬门槛节点，如二次复核未过不得装车。
4. 异常中心
   结构化记录数量异常、单证异常、板位异常、时效异常、运输异常、质量异常。
5. KPI 面板
   查看 12 小时完成率、装车准确率、POD 闭环率、异常闭环时长。
6. 班组与资源协同
   查看班次、人手、卡车、装车 session、司机 / 车牌登记。

### 5.3 样板站建议

优先以 `MME` 为样板站做第一版后台，因为说明书已经提供了最成熟的 SOP。

## 7. 原型里的重点数据对象

本次页面已围绕以下对象组织内容：

- `Order`
- `Shipment`
- `AWB`
- `ULD / PMC`
- `Truck`
- `Flight`
- `POD`
- `Delivery`
- `Customer`
- `Contract`
- `Event`

## 8. 下一步建议

如果继续推进到 PRD，可按以下顺序细化：

1. 冻结试点站点与试点链路
2. 冻结主键体系与主数据归属
3. 先写节点状态机，再写页面交互
4. 先定异常字典与 SLA，再做报表
5. 先做 `MME` 样板站，再复制到其他站点
