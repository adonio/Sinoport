## 6. 下一年度规划

下一年度只保留真正还需要推进的部分：

### 6.1 真实业务数据接入深化

- 扩大真实数据导入范围
- 稳定数据校验、回写、审计和回放
- 让 demo 过渡层逐步服务于真实业务，而不是额外的前端依赖

### 6.2 运维与安全收口

- 将 `AUTH_TOKEN_SECRET` 切换为真正的 Cloudflare secret
- 收紧 demo 账号与本地 bootstrap 逻辑
- 明确 staging / production 的发布和回归边界

### 6.3 报表与多站点治理深化

- 继续稳定日报与异常日报
- 强化站点模板、主数据、角色与配置治理
- 降低新站点接入成本

### 6.4 Agent 生产化验证

- 让 `Station Copilot` 和 `Document Agent` 在真实业务中持续可用
- 补齐工具、审计、边界和告警
- 验证 Agent 是否真正提升作业效率

### 6.5 仓库与构建产物治理

- 处理 `admin-assets/` 和静态路由产物污染工作区的问题
- 将构建产物改成更清晰的 CI-only 或临时目录策略

## 7. 主要风险

### 风险 1：生产 secret 未完全收口

- 影响：生产环境安全边界不清晰
- 对策：优先完成 Cloudflare secret 切换

### 风险 2：真实业务数据质量波动

- 影响：导入链和报表链的稳定性下降
- 对策：保持审计、回放和回滚能力

### 风险 3：Agent 价值不稳定

- 影响：产品化后仍可能停留在演示层
- 对策：只保留能落地到真实作业的工具和动作

### 风险 4：构建产物继续污染仓库

- 影响：日常开发与发布节奏持续受扰
- 对策：尽快明确产物管理策略
