## 2. 决策原则

1. 每个月只记录主 agent 必须做出的关键决策。
2. 决策一旦冻结，不在下一个月被隐式推翻。
3. 若未来需要改动，必须在新的版本文档中显式覆写。

## 3. 月度决策

### M1 生产安全收口

- 决策 `M1-D1`：保留 demo 账号，但仅允许 `local-only` 使用。
- 决策 `M1-D2`：`AUTH_TOKEN_SECRET` 与同级安全变量必须迁入 Cloudflare / GitHub secrets，不允许继续作为生产明文配置。
- 决策 `M1-D3`：`X-Debug-*` 与 `demo-token` 统一降级为本地开发开关，非 local 环境视为不通过。

### M2 构建产物治理

- 决策 `M2-D1`：默认方向采用“`CI / 临时目录生成`”，而不是继续把构建产物当源码日常提交。
- 决策 `M2-D2`：构建产物治理要纳入发布前检查项，不能只靠 `.gitignore`。
- 决策 `M2-D3`：发布脚本默认写入 `.generated/admin-static`，`site-dist` 只作为最终装配物，不再反写仓库根目录静态路由。

### M3 发布与运维基线

- 决策 `M3-D1`：发布顺序固定为 `staging -> production`。
- 决策 `M3-D2`：发布后校验固定化，视为交付链的一部分，而不是可选项。
- 决策 `M3-D3`：D1 / R2 必须有明确备份、恢复和回滚口径。
- 决策 `M3-D4`：版本口径固定为 `git SHA + 可选 tag`，并直接并入 `healthz` 返回，不单独拆 `/version`。
- 决策 `M3-D5`：D1 / R2 在本月先做到可执行 Runbook，不额外扩展自动备份脚本。

### M4 真实数据导入一期

- 决策 `M4-D1`：第一批只聚焦一条最小正式导入链，不同时扩面。
- 决策 `M4-D2`：`demo_datasets` 只允许作为后端过渡与回放层，不再回到前端主读源。
- 决策 `M4-D3`：导入验收必须同时具备审计、回放和重试能力。
- 决策 `M4-D4`：`request_id / Idempotency-Key` 冻结为正式幂等键，相同键的重复导入只允许返回 `replayed`，不重复写业务对象与正式审计。

### M5 单站真实试运行

- 决策 `M5-D1`：试运行一次只允许 `1` 个真实站点。
- 决策 `M5-D2`：试运行期间优先冻结非必要变更，保证问题可归因。
- 决策 `M5-D3`：验收结果必须沉淀成站点试运行 SOP。
- 决策 `M5-D4`：在真实试运行站点、责任人和值班安排未冻结前，`M5` 不得进入实施，只能维持 `Blocked`。
- 决策 `M5-D5`：未明确 `staging` 预演还是 `production` 直跑前，主 agent 不得擅自假设执行环境。
- 决策 `M5-D6`：当前首轮单站试运行直接采用 `production`，不增加单独 `staging` 预演阶段。
- 决策 `M5-D7`：当前首个真实试运行站点固定为 `MME`，不额外引入第二个站点。

### M6 数据质量治理

- 决策 `M6-D1`：质量治理先建立可执行的分级与追踪，再逐步加严规则。
- 决策 `M6-D2`：高严重级数据问题允许进入默认阻断候选集，但以规则冻结为前提。
- 决策 `M6-D3`：日报、对象链、审计链必须能够互相回查问题来源。
- 决策 `M6-D4`：质量问题必须成为正式数据库对象，不允许继续只留在日志或日报推断里。
- 决策 `M6-D5`：`VALIDATION_ERROR` 这类失败导入也必须进入 `import_requests` 账本，不能遗漏失败样本。
- 决策 `M6-D6`：当前默认阻断候选集固定为 `DQ_IMPORT_REQUEST_FAILED`、`DQ_AWB_MISSING_FLIGHT`、`DQ_AWB_MISSING_SHIPMENT`、`DQ_TASK_MISSING_RELATED_OBJECT`。
- 决策 `M6-D7`：`DQ_SHIPMENT_WITHOUT_AWB` 与 `DQ_TRIAL_FLIGHT_MISSING_AUDIT` 当前阶段保留为提醒型问题，不进入默认阻断候选集。
- 决策 `M6-D8`：平台/站点日报必须同时暴露 `qualitySummary` 与 `qualityChecklist`；`qualityChecklist.gate_status` 只允许 `clear / warning / blocked`。
- 决策 `M6-D9`：`M6` 验收以“问题能沉淀、能追踪、能回灌”为准，本月不额外引入显式 issue close 工作流。

### M7 出港深化

- 决策 `M7-D1`：`loaded / manifest finalize / airborne` 必须采用统一失败语义和状态推进规则。
- 决策 `M7-D2`：`manifest finalize` 之后默认进入对象锁定语义，后续修改需显式规则支持。
- 决策 `M7-D3`：出港异常不只要能阻断，还必须能恢复并留痕。
- 决策 `M7-D4`：正式验收链固定为“命中阻断型出港异常 -> 通过异常恢复接口解除阻断 -> 再继续 loaded / manifest finalize / airborne”，不再允许直接跳过阻断异常。
- 决策 `M7-D5`：航班页负责表达出港动作门槛，AWB 页只负责表达异常恢复状态；文件门槛继续单独保留在文件门槛摘要，不与恢复摘要混用。
- 决策 `M7-D6`：出港日报与对象联动本月只收口到站点报表概览层，固定最小字段集为“动作进度、阻断原因、最近审计、对象跳转”，不提前扩成独立日报家族。

### M8 报表稳定化

- 决策 `M8-D1`：三类日报必须先统一最小核心指标口径，再扩展维度。
- 决策 `M8-D2`：数据质量问题必须显式暴露到报表层，而不是只留在治理页。
- 决策 `M8-D3`：多站点对比先固定最小对比集，避免口径扩散。
- 决策 `M8-D4`：平台报表页与货站报表页默认只读取 `daily` 接口，概览样例不再作为默认读源。
- 决策 `M8-D5`：日报页默认展示区固定为“生成锚点 + 核心指标 + 质量摘要 + 质量检查表”；班次、文件、动作摘要继续保留为扩展区块。
- 决策 `M8-D6`：日报计算口径固定为“按 `reportDate` 的日终锚点执行全量重算”，不引入增量刷新。
- 决策 `M8-D7`：补算范围固定为“同一作用域 + 同一日期”，不允许跨日补算。
- 决策 `M8-D8`：日报必须显式返回 `refreshPolicy` 与 `traceability` 元数据，不能只写在文档里。
- 决策 `M8-D9`：最小多站点对比集固定为“真实主站 + 模板对照站”，当前优先采用 `MME + RZE`，不把模板对照站视为真实试运行日报。

### M9 多站点复制

- 决策 `M9-D1`：复制包必须区分“强制一致项”和“站点可覆盖项”。
- 决策 `M9-D2`：复制流程必须具备回放与回滚口径，才能视为可运营。
- 决策 `M9-D3`：多站点接入验收以“无需逐页手工配置”为核心标准。
- 决策 `M9-D4`：本月先固化模板包、接入 SOP 与回滚口径，不提前实施第二真实站点上线。
- 决策 `M9-D5`：当前主样板站固定为 `MME`，模板对照站固定为 `RZE`；`RZE` 仅用于模板对比，不视为第二真实试运行站。
- 决策 `M9-D6`：质量问题在 `M9` 先维持“治理检查 warning 优先、接入验收按 checklist 决定是否阻断”，不在本月扩大运行时阻断范围。
- 决策 `M9-D7`：回滚口径固定为“模板 + 配置级回滚”，不允许只回滚模板绑定。
- 决策 `M9-D8`：`M9` 必须同时交付模板包定义、接入 SOP、接入检查清单和回滚口径，不接受只交文档说明。
- 决策 `M9-D9`：`warning` 允许进入接入完成态，但必须在正式验收记录中完成人工确认。
- 决策 `M9-D10`：`M9` 的回滚对象只包含模板绑定与站点配置，不回滚已落库真实业务数据、审计事件和已形成日报结果。
- 决策 `M9-D11`：`comparisonType` 只允许 `actual / template` 两种，不引入第三类站点状态。
- 决策 `M9-D12`：`acceptanceDecision` 固定为 `Accepted / Refine / Blocked` 三态。
- 决策 `M9-D13`：`rollbackScope` 必须出现在正式验收记录模板中，不能省略。
- 决策 `M9-D14`：`RZE` 可以出现在治理对比与验收文档示例中，但仍只作为模板对照站，不进入真实接入流程。
- 决策 `M9-D15`：`M9` 最终月度验收标准固定为“能重放、能对比、能回滚、能归档”。

### M10 Station Copilot 生产验证

- 决策 `M10-D1`：本月聚焦生产价值验证，不新开 Agent 产品线。
- 决策 `M10-D2`：验证优先采用真实对象链下的只读或建议型场景。
- 决策 `M10-D3`：是否继续投资，以量化价值证据为准，而不是主观好用感。
- 决策 `M10-D4`：本月固定为“只读 + 建议型”验证，不允许自动执行写动作。
- 决策 `M10-D5`：生产价值验收至少同时包含 `建议采纳率`、`查询耗时改善`、`人工操作减少量` 三个量化指标。
- 决策 `M10-D6`：若出现越权读取、越权建议或审计缺失，立即收紧回纯只读模式后再继续验证。
- 决策 `M10-D7`：`W10-01` 验证对象链固定为 `MME` 下的 `Station / Flight / AWB / Shipment / Exception / Document`，其中主样本固定为 `SE803`、`SE913`、`436-10358585`、`436-10357583`、`SHIP-IN-436-10358585`、`SHIP-OUT-436-10357583`、`EXP-0408-001`。
- 决策 `M10-D8`：`RZE` 在 `M10` 仍只作为模板对照背景，不进入真实 Copilot 验证结论。
- 决策 `M10-D9`：`W10-01` 的工具白名单固定为 `get_flight_context`、`get_outbound_flight_context`、`get_outbound_waybill_context`、`get_station_shipment_context`、`get_station_exception_context`、`get_station_document_context`、`list_blocking_documents`、`list_open_exceptions`、`get_object_audit`。
- 决策 `M10-D10`：`request_task_assignment` 不进入 `M10` 验证白名单；本月按“隐藏/不展示写入口”口径处理，而不是仅禁用执行。
- 决策 `M10-D11`：`W10-01` 样本量先冻结为 `MME + 固定对象样本集` 的最小集，不额外扩面；是否加样本只在 `W10-03` 根据证据再决定。
- 决策 `M10-D12`：`W10-02` 的失败样本口径固定为 `401 / 403 / 404 / 400` 四类，并要求同输入重放后错误码稳定。
- 决策 `M10-D13`：`W10-02` 验收必须同时证明三条链存在：`agent_sessions / agent_messages / agent_runs` 会话链，`available_tools / execute` 工具链，`audit_events / state_transitions` 审计链。
- 决策 `M10-D14`：`request_task_assignment` 在 `M10` 必须从 UI 和会话推荐面中彻底隐藏，不接受“仅禁用执行”的口径。
- 决策 `M10-D15`：`W10-03` 样本量继续冻结为 `MME + 固定对象样本集`，不额外扩充第二真实站点或额外对象类型。
- 决策 `M10-D16`：`建议采纳率` 的最小通过阈值冻结为 `>= 60%`，且至少满足 `5` 个有效会话、`3` 个采纳会话。
- 决策 `M10-D17`：`查询耗时改善` 的基线统一采用“无 Copilot 的手工路径”，以首次得到可用结论的总耗时为准。
- 决策 `M10-D18`：`人工操作减少量` 以 `页面跳转数` 作为主口径，`独立查询数` 作为辅助口径。
- 决策 `M10-D19`：`MME + 固定对象样本集` 冻结为后续回归基线，除非新开月份重新扩样。
- 决策 `M10-D20`：`Station Copilot` 在本阶段的最终定位固定为“继续投入，但仅限受控只读 + 建议型能力”。
- 决策 `M10-D21`：`M10` 通过后不自动进入 `M11`，是否开启 `Document Agent` 需要显式新指令。

### M11 Document Agent 生产验证

- 决策 `M11-D1`：聚焦真实单证对象链，不扩展新文档类型。
- 决策 `M11-D2`：验证优先采用只读与建议型动作，避免未验证自动执行扩散。
- 决策 `M11-D3`：继续投资标准同样以量化价值和审计完整度为准。
- 决策 `M11-D4`：本月完全沿用 `M10` 的“只读 + 建议型、无自动写动作”原则，不开放任何单证写动作工具。
- 决策 `M11-D5`：验证范围冻结为 `MME + 固定单证样本集`，允许补充一组失败样本作为正式回归基线。
- 决策 `M11-D6`：白名单工具固定为单证上下文读取、`NOA/POD` 状态读取、关联 `Task / Exception / Audit` 读取，不包含任务分派或状态推进入口。
- 决策 `M11-D7`：最小价值通过阈值沿用 `M10`，即采纳率 `>= 60%`、有效样本 `>= 5`、采纳样本 `>= 3`。
- 决策 `M11-D8`：若 `Document Agent` 只证明查询提效而未证明单证建议价值，则结论收缩为“只读查询工具”，不扩面。
- 决策 `M11-D9`：`DOC-POD-TRK-0406-018` 冻结为 `M11` 的唯一 `POD` 主样本，本月不再补第二个 `POD` 失败样本进入正式基线。
- 决策 `M11-D10`：`AWB-436-10358585` 冻结为 `NOA/POD` 统一锚点，本月不再引入第二张 AWB。
- 决策 `M11-D11`：`get_station_shipment_context` 明确排除出 `M11` 白名单，避免范围回到履约链全景。
- 决策 `M11-D12`：`W11-01` 通过后，若最终只证明“查询提效”而未证明“单证建议价值”，则直接按 `M11-D8` 收缩为“只读查询工具”。
- 决策 `M11-D13`：`W11-02` 的硬门槛是 `/api/v1/agent/tools` 只能暴露 `W11-01` 冻结的 5 个白名单工具，出现任何 Flight/Shipment/写动作工具都判 `Refine`。
- 决策 `M11-D14`：`get_object_audit` 的最小要求固定为 `Document + AWB` 审计非空，`Exception` 至少能回读链路，不接受“接口 200 但空结果”。
- 决策 `M11-D15`：`DOC-CBA-SE803` 也必须完整跑过会话链，但价值判断主样本仍以 `DOC-MANIFEST-SE803` 与 `DOC-POD-TRK-0406-018` 为主。
- 决策 `M11-D16`：允许单个 `Document` 样本的 `related_tasks` 或 `related_exceptions` 为空，但整体样本集必须证明 `Document / Task / Exception / Audit` 链可打通。
- 决策 `M11-D17`：`W11-03` 的真实生产场景固定为 6 个，其中主价值判断场景聚焦 `Manifest gate` 与 `POD release`，其余场景作为辅助证明。
- 决策 `M11-D18`：`pageJumpReduction` 不要求 `5/5` 场景全为正，但要求大多数场景为正且总量为正。
- 决策 `M11-D19`：`建议采纳` 必须由人工显式标记，不允许脚本自动推断。
- 决策 `M11-D20`：若 `NOA/POD` 主场景证明了建议价值，而 `CBA` 仅证明查询价值，则仍可判定“继续投入”，但范围只限 `NOA/POD` 单证链。
- 决策 `M11-D21`：`NOA/POD` 主场景至少需要 `2` 个被人工显式标记为采纳，才允许判定“继续投入”。
- 决策 `M11-D22`：`CBA` 样本只作为辅助证据，不参与“继续投入”的主价值判断。
- 决策 `M11-D23`：若 `NOA/POD` 价值成立而 `Task/Exception` 关联解释仍偏弱，允许先判“继续投入但限缩范围”。
- 决策 `M11-D24`：`M11` 通过后不自动进入 `M12`，是否开启年度复盘需要显式新指令。

### M12 年度复盘与二期规划

- 决策 `M12-D1`：下一年度只保留三条主线：
  - 真实业务接入
  - 治理与运营稳定化
  - 生产化验证深化
- 决策 `M12-D2`：年度复盘必须同时输出管理层版和执行层版。
- 决策 `M12-D3`：若无法回答“解决了什么、留下了什么”，则不允许冻结下一年度路线。
- 决策 `M12-D4`：已收到显式继续指令，允许从 `M11` 切换进入 `M12`，但当前只允许启动 `W12-01 年度复盘取证范围冻结`。
- 决策 `M12-D5`：`M12` 在 `W12-01` 阶段不得提前细化下一年度实施动作，只能先冻结复盘证据范围、输入边界与输出物清单。
- 决策 `M12-D6`：下一年度的管理层版与执行版文档必须分开冻结，不允许只保留一个版本。
- 决策 `M12-D7`：下一年度继续投入只保留三条主线：真实业务接入深化、治理与运营稳定化、生产化验证深化。
- 决策 `M12-D8`：`Station Copilot` 下一年度继续保留受控只读 + 建议型范围；`Document Agent` 下一年度仅保留 `NOA/POD` 单证链验证范围。

## 4. 当前结论

- `M1` 至 `M12` 的关键决策已冻结并进入实施结果，本轮主计划已收口完成。
