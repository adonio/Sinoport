# Sinoport OS 第一批前端 Demo QA / UAT 清单

## 1. 使用说明

本清单只针对第一批前端 demo。

约束：

- 不验证任何真实后端接口
- 不验证真实登录、真实上传、真实持久化
- 只验证前端页面、mock 状态、交互闭环和演示链路

## 2. 冒烟清单

### 2.1 平台端

- [ ] 打开 `/platform/operations`，默认进入运行态势中心
- [ ] 运行态势中心顶部 KPI、告警、待处理动作、站点健康度矩阵正常显示
- [ ] 打开 `/platform/rules`，能看到规则模板、证据要求、标准场景编排
- [ ] 打开 `/platform/master-data`，能看到主数据口径、接口治理、模拟导入日志
- [ ] 平台菜单、搜索、workspace 入口与页面命名一致

### 2.2 货站端

- [ ] 打开 `/station/dashboard`，能看到阻塞、待复核、二次转运和航班概览
- [ ] 打开 `/station/inbound`，能看到进港状态链、文件门槛、阻断提示
- [ ] 打开 `/station/outbound`，能看到出港状态链、文件门槛、阻断提示
- [ ] 打开 `/station/shipments`，能看到统一的 Shipment / AWB 目录
- [ ] 打开 `/station/documents`，能看到文件台账、进港/出港放行门槛、指令模板
- [ ] 打开 `/station/tasks`，能看到任务池、待复核/待升级、标准场景编排
- [ ] 打开 `/station/exceptions`，能看到异常 -> 阻断任务 -> 恢复动作

### 2.3 PDA 端

- [ ] 打开 `/mobile/login`，能进入移动端
- [ ] 打开 `/mobile/inbound`
- [ ] 进入任一航班后，概览页能看到统一任务卡
- [ ] 拆板 / 理货页能看到统一任务卡、证据要求、阻断说明
- [ ] 组托页能看到统一任务卡
- [ ] 装车计划页和装车执行页都能看到统一任务卡

## 3. 状态机验收清单

### 3.1 进港主线

- [ ] 航班详情、提单与履约链路、任务中心、单证中心对同一对象的状态表达一致
- [ ] `理货完成前不得发送 NOA` 的规则在平台、货站、PDA 三端文案一致
- [ ] `POD 双签前不得 Closed` 的规则在提单链路、异常中心、单证页表达一致

### 3.2 出港主线

- [ ] 出港页能表达 `预报 -> 接收 -> 主单 -> 装载 -> 飞走 -> Manifest 回传`
- [ ] `Manifest 未冻结不得飞走归档` 的规则在规则页、单证页、异常页表达一致

### 3.3 任务状态

- [ ] 任务池中的状态标签与对象详情中的任务状态一致
- [ ] 待复核、待升级、阻塞三类任务都能在任务中心中找到对应示例

## 4. 文件门槛验收清单

### 4.1 进港

- [ ] `CBA / Manifest / Handling Plan` 缺失时，进港放行门槛显示阻断
- [ ] `理货记录 / 差异结论` 缺失时，NOA 门槛显示待处理
- [ ] `POD 双签 / Delivery Record` 缺失时，Closed 门槛显示阻塞

### 4.2 出港

- [ ] `Origin POD / AWB / CBA` 缺失时，出港收货到组板门槛能表达阻断逻辑
- [ ] `FFM / UWS / Manifest` 缺失时，Loaded -> Airborne 门槛能表达阻断逻辑

### 4.3 跨页一致性

- [ ] 同一个门槛在平台规则页、货站单证页、货站任务页文案一致
- [ ] 同一个门槛在 Shipment 详情页中能找到对应文件或异常说明

## 5. PDA `MME` 进港链路演示清单

### 5.1 进入链路

- [ ] 登录移动端
- [ ] 选择进港
- [ ] 进入 `SE803` 或其他 `MME` 样板航班

### 5.2 航班概览

- [ ] 航班概览页展示统一任务卡
- [ ] 能看见航班当前节点、货量、提单基础详情

### 5.3 拆板 / 理货

- [ ] 进入 `breakdown` 页
- [ ] 页面顶部存在统一任务卡
- [ ] 任务卡显示 SLA、证据要求、阻断说明
- [ ] 扫码提单后能纳入统计
- [ ] 航班外提单会先弹确认，再决定是否纳入

### 5.4 组托

- [ ] 进入 `pallet` 页
- [ ] 页面顶部存在统一任务卡
- [ ] 新建托盘逻辑可用
- [ ] 不同 consignee 混托时能表达阻断

### 5.5 装车

- [ ] 进入 `loading` 页
- [ ] 装车计划页有统一任务卡
- [ ] 创建装车计划页有统一任务卡
- [ ] 装车执行页有统一任务卡
- [ ] 能表达 `车牌 / Collection Note / 核对员缺失时不得完成装车`

## 6. 平台 -> 货站 -> PDA 演示顺序

### 6.1 平台

1. 进入运行态势中心
2. 说明当前全网风险与阻塞链路
3. 进入规则与指令引擎，说明任务模板、证据要求、硬门槛
4. 进入主数据与接口治理，说明这批仅做前端 contract

### 6.2 货站

1. 进入货站看板
2. 进入进港管理，看状态链与文件门槛
3. 进入提单与履约链路，打开一条 Shipment 详情
4. 从 Shipment 详情跳转到单证中心、任务中心、异常中心
5. 进入异常中心，说明阻断任务和恢复动作

### 6.3 PDA

1. 登录移动端
2. 进入 `MME` 进港航班
3. 从航班概览进入理货、组托、装车
4. 说明统一任务卡如何承载动作、证据、阻断

## 7. 通过标准

满足以下条件即可判定第一批完成：

- 平台、货站、PDA 三端主入口可正常打开
- 第一批主路径页面都能解释“为什么能做 / 为什么不能做”
- `MME` 进港样板链路可完整演示
- 有本清单作为正式验收依据
